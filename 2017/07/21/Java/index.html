二、一切都是对象

2.1 对象存储

对象引用 存储在 栈(stack) 中，位于RAM(Random Access Memory, 随机存取存储器)，但通过 堆栈指针 可以从处理器那里获得直接支持。创建程序时，Java系统必须知道存储在栈内所有项的确切生命周期，以便上下移动堆栈指针；Java对象 存储在 堆(heap) 中，位于通用的RAM区。在堆里分配存储有很大的灵活性，编译器不需要知道存储的数据在堆里存活多长时间，自然用堆进行存储分配和清理可能比用栈进行分配需要更多的时间。在对一个对象进行操作时，真正操作的是 对对象的引用，故赋值时是将 引用 从一个地方复制到另一个地方。

用new时会将对象存储在“堆”里，但创建一个小的、简单的的变量时会造成很大的碎片化，因此对于 基本类型，Java会创建一个并非是引用的自动变量，直接存储值，并放在 栈 中，这会更加有效。基本类型存储了实际的值，而并非指向一个对象的引用，所以在赋值时，是直接将一个地方的内容复制到了另一个地方。
清楚了基本类型和引用存储在stack中，对象内容存储在heap中。接下来一个著名的问题是：Java是值传递还是引用传递？ 参考链接：Does Java pass by reference or pass by value?和Java 到底是值传递还是引用传递？。
这被称为浅复制(shallow copy)，即复制对象的引用，而不是对象本身的拷贝。作为参数进行函数调用时，会在 stack 中会生成原值的一个 副本，本质上只会操纵stack区中的值。在函数 作用域(scope) 内，对于基本类型来说，就是直接copy值赋给参数，对于引用类型来说，就是copy对象引用地址，这时就会有两个引用指向一个对象空间。故在写 交换函数（swap） 时，最好对对象内的字段进行交换，而不是通过=符号直接赋值交换，不然很容易出现bug。
基本类型放在栈中，所以Java要确定每种基本类型所占存储空间的大小，它们的大小不像其他大多数语言随机器硬件架构的变化而变化。一共有9种基本类型(单位为bit)：void、boolean；byte - 8、char - 16；short - 16、int - 32、long - 64；float - 32、double - 64。所有的数值类型都有正负号，所以没有无符号(unsigned)的数值类型。它们都有对应的 包装器类，其在堆中创建非基本对象。Java的 自动包装(wrapper)功能 会自动把基本类型和包装器类型转换。
BigInteger和BigDecimal分别支持任意精度的整数和小数，运算更加复杂，故运算速度比较慢。以速度换取了精度。

2.2 对象销毁

Java使用 垃圾回收器(GC, Garbage Collection) 来监视用new创建的所有对象，对于不再引用的对象，会释放这些对象的内存空间。

2.3 类的创建

在Java中所做的全部工作就是定义类，产生类的对象，以及发送消息给这些对象。在类中可以设置两种元素：字段(Field) 和 方法(Method)。字段可以是对象引用也可以是基本类型，若类的字段是 基本数据类型，即使没有进行初始化，Java也会确保它获得一个默认值，但确保初始化的方法并不适用于 局部变量(即并非类字段，方法内变量)。
在定义包名时，Java设计者希望程序员反过来使用自己的Internet域名。
通常来说，当创建类时，就是在描述那个类的外观和行为。执行new来创建对象时，数据存储空间才被分配，其方法才能被外界调用。当希望字段或方法不与包含它的类到的任何对象关联在一起，即使没有创建对象，也能够使用字段或方法，可以用 static 关键字来标识，这样字段和方法是作为整个类，而不是类的某个特定对象而存在的。

三、操作符

  • 整数除法会直接去掉结果的小数位，而不是四舍五入地圆整结果。
  • 对于 前缀(prefix) 递增或递减(如 ++a 或 –a)，会先执行运算，再生成值；对于 后缀(suffix) 递增或递减(如 a++ 或 a–)，先生成值，再执行运算。
  • ==和=!比较的是对象引用，比较对象内容是否相同使用equals()，其本质是比较两个对象的 hashcode。而基本类型直接使用==和=!即可。
  • 当进行 逻辑运算 时，会出现 短路(short circuit)，即一旦能够明确无误地确定整个表达式的值，就不在计算表达式余下部分了，这样就会获得潜在的性能提升。
  • 按位操作符和移位操作符用来操作 整数基本类型 中的单个 比特bit，即二进制位。它来源于C语言面向底层的操作，会直接操作硬件，设置硬件寄存器内的二进制位。

四、控制流程

switch(integral-selector)

其中integral-selector(整数选择因子)必须是 int 或 char 那样的整数值，因 enum 本质上其实就是整数的映射，故enum也可以和switch协调工作。

五、初始化顺序

用new创建对象时，会调用相应的 构造器(constructor)，可确保每个对象都会得到初始化。通过 重载(overload) 可以编写默认构造器和带有参数的构造器，方法名和参数列表(合起来被称为“方法签名”)唯一地标识出某个方法，重载就是方法名一致而参数列表不同。为了减少代码冗余，可以通过 this关键字 在构造器中调用其他构造器，this表示对当前对象的引用，用于内部方法的调用。
new创建对象的初始化顺序，也就是程序执行的顺序为：Static Field(或静态块) -> No Static Field(或 代码块) -> Constructor。如果JVM已经创建了一个类的对象，那么再次new改类对象时，静态变量就不会初始化了，初始化的顺序就变成了非静态变量 -> 构造器。原因是JVM在加载对应类的.class文件是创建一个 Class对象，静态初始化只在Class对象首次加载时进行一次。

六、访问权限控制

JVM在编译完一个.java文件后，会在该文件中每个类生成 可执行的.class文件，Java解释器负责这些文件的查找、装载和解释。当编译器遇到使用import引入一个类或包时，解释器就会遍历环境变量 CLASSPATH 中包含的目录，即用作查找.class文件的根目录，从已编译的文件中找出名称相符者。当使用IDE(如IntelliJ IDEA 或 Eclipse)手动引入依赖，或使用构建工具(如Maven 或 Gradle)自动引入依赖时，本质上也是把jar包放在了CLASSPATH包含的路径里。
PS：不要错误的认为Java总是将当前目录当作查找行为的起点之一，如果CLASSPATH之中缺少了一个“.”作为路径之一的话，Java就不会查找那里。

访问控制权限的LEVEL，权限从大到小：public、protected、包访问权限、private。它们之间是包含关系。
类的访问权限：类只有public 和 包访问 两个访问权限(内部类可以是protected或private)。
如果不希望任何人对该类拥有访问权限，可以把所有构造器指定为private，从而阻止任何人创建该类的对象，这样就只能在该类的static成员内部创建。这就是设计模式(Design Pattern)中的 Singleton(单例)，这样保持始终只能创建该类的一个对象，有且只有一个。

七、代码复用

当子类 继承 父类后，初始化过程是从 父类 向外扩散的，依次初始化子类。
代理(Delegated) 是继承和组合的中庸之道，即将一个对象作为所构造类的成员变量(就像组合)，同时在新类中暴露了该成员对象的所有方法(就像继承)。之所以不直接使用继承，大多数情况下是所构造的类和该成员对象不是同一概念，更像包含关系都又很相似。比如手机代理商代理了iphone，代理商帮助苹果卖手机(继承了这个行为方法，都是卖手机)，但不表示代理商就是手机制造商(故不能继承整个类)。

7.1 final关键字

当final作用域变量时，对于基本类型，数值恒定不变，对于对象引用，引用恒定不变，但是对象本身内容是可以被修改的。final的本质是：只能使stack中的值保持不变，不影响heap中的对象值。

7.2 继承结构类的初始化顺序

当父类和子类中都有静态成员变量和普通成员变量时，在创建对象初始化前，所以的类都必须先被JVM加载完成(在这个过程中会初始化static变量)，而初始化总是先从父类开始到子类。故总的初始化顺序为：父类static field -> 子类 static field，这是类加载完毕，开始创建对象， 父类 plain field, Constructor -> 子类 plain field, Constructor。

八、多态(Polymorphism)

多态也称为 运行时绑定，它的作用是消除 类型 之间的耦合关系，这样编写方法时可以仅接收父类作为参数，而不是特殊的子类。
将方法和具体对象关联起来称为 绑定。若在程序执行前进行绑定，称为 编译期绑定，面向过程语言默认使用编译期绑定，如C只有这种绑定方式。而Java采用的是的运行时绑定，即 在运行时根据对象的类型 进行绑定。如果一种语言采用了运行时绑定，编译期就一直不知道对象的类型，就必须在对象中安置某种“类型信息”，以便在运行时能判断对象的类型(RTTI)。
经典的例子就是 几何形状(shape)。圆(Circle)、正方形(Square)、三角形(Triangle)都继承形状(Shape)，只需调动父类Shape的方法，在运行时能根据当时对象的类型，调用正确的子类执行正确的代码。
PS: 只有普通的方法调用可以是多态的，成员变量访问操作在编译期解析，因此不是多态的。

十、内部类(Inner Class)

内部类就是将一个类定义在另一个类的内部，是一种代码隐藏机制。
当一个外围类对象创建了一个内部类对象时，内部类对象就会捕获一个指向那个外围类对象的引用，内部类会链接到它的外围对象(enclosing object)，拥有外围类的所有元素的访问权。

  1. 局部内部类(Local Inner Class)：在一个 方法的作用域内 而不是在其它类的作用域内创建的一个内部类。局部内部类只能在方法的作用域内访问到，方法作用域外是不可见的。
  2. 匿名内部类(Anonymous Inner Class)：创建一个 继承 自父类的一个匿名类对象，本身没有名字，通过new表达式返回的引用被自动 向上转型(upcast) 为对父类的引用。
  3. 嵌套类：内部类被声明为static。这样创建嵌套类的对象，就不需要外围类对象，同样这个嵌套类也不能访问非静态的外围类对象了。

闭包(closure) 是一个可调用对象，它记录了创建它的作用域的一些信息。在Java中，内部类就是面向对象的闭包，因为它有权操作外围类的所有元素，便允许 回调(callback)，借用内部类在稍后的某个时刻调用初始的对象。

十三、Java正则表达式

在 正则表达式(Regular Expression) 中，Java对 反斜线\ 的处理和其他语言不同。在其他语言中，\\就表示插入一个普通反斜线，没有特殊意义；而在Java中，\\表示插入正则表达式的反斜线，有特殊意义，\\\\表示插入普通的反斜线。
单位：?–0或1；*–0~∞；+–1~∞。
.表示任意字符。
量词：贪婪型(默认)，最大范围匹配变量；非贪婪型(?)，最小范围匹配变量。

十四、类型信息(Type)

Java如何让我们在运行时识别对象和类的信息。主要有两种方式：一种是传统的 运行时类型识别(RTTI)，JVM会帮我们做这些事；另一种是 反射机制(reflect)，它们本质上使用Class对象，它包含了与类有关的信息。每当编译了一个新类就会产生一个Class对象，在程序运行期间，所有的类在第一次被使用时，类加载器会把该类对应的Class对象动态加载到JVM中，一旦某个类的Class对象被加载到内存中，就可以被用来创建这个类的所有对象。
通过反射可以调用一个类的所有方法(包括private方法)，可以获取并修改所有的成员变量(包括private字段)，可以说，在反射面前，没有权限访问这个概念。不过即使是反射也不能 修改final变量 (依然可以读取)。

十五、泛型(Generic)

为了兼容已经基于Java旧版本实现的代码，当Java SE5 实现的泛型时，任何具体的类型信息都被 擦除 了，即 在泛型代码内部，无法获得任何有关泛型参数类型的信息。因此List和List在运行时事实上是相同的类型，都被擦除成了它们原生的类型，即List。
在实际上场景中，为了调用想要的方法，可以使用extends关键字来给定泛型类的边界，以此来告知编译期只能接受这个边界的类型。不过当获取到这个对象的引用时，还是可以向下转型成具体的类型。

十六、hashcode和equals

我们常用的String类的equals()比较的是对象的值，是因为Java帮我们实现了对应的hashcode()，实际上默认的Object.equals()依然比较对象的地址，和==是一样的。
首先，使用hash的目的就是想要用一个对象来快速的查找一个对象，更关注生成速度，而不是唯一性，故hashcode必须基于 对象的内容 生成(请求机器硬件进行地址比较是比较慢的)。
故在选择散列函数时是十分重要的，既要保证 处理速度 (对现代的CPU来说，除法和取模运算是最慢的操作)，又要保证 冲突 不能太多(解决冲突 再散列 的开销也不小)。好的hashcode()应该产生分布均匀的散列码。

十七、fail-fast

Java容器采用 快速报错(fail-fast) 机制，来防止程序的不同部分修改同一容器的内容。它使用 expected期望的计数或容器长度 和 当前实际的计数或长度 来比较，发现不一致就报ConcurrentModificationException异常。比如当迭代遍历容器时，另一个线程在增删改该容器的某个对象，根据比较期望值和实际值不一致，能立马发现不一致变化，从而快速抛出异常，而不是使用复杂算法事后检查问题。

十八、持有引用

如果想继续持有对某个对象的引用，希望以后还能够访问到该对象，但是也希望能够允许垃圾回收器释放它，这时就应该使用Reference对象。这样就可以继续使用该对象，而在内存消耗殆尽的时候又允许释放该对象。SoftReference、WeakReference、PhantomReference有强到弱，对应不同级别的“可获得性”。SoftReference用于实现内存敏感的高速缓存；WeakReference用于实现“规范映射”；PhantomReference用于调度回收前的清理工作。

十九、I/O 系统

I/O库常使用 流(stream) 这个概念，代表任何有能力产出数据的数据源对象或是有能力接收数据的接收端对象。Java中的I/O类分为 Input输入(根类Inputstream或Reader) 和 Output输出(根类OutputStream或Writer)，Java中都是通过叠合多个对象来提供所期望的功能，使用了 装饰器模式。其中Inputstream和Outputstream用来处理字节，而Reader和Writer用来处理字符，如果想把这两个结构层次结合起来，就会用到 适配器模式。
一个常见的程序化任务就是读取文件到内存，修改，然后在写出。

NIO - 新I/O

NIO作为新的Java I/O类库，其目的在于提高速度，可以快速移动大量数据，其所使用的结构更接近于操作系统执行I/O的方式：通道和缓冲器。应用程序并不会和通道交互，只是和缓冲器交互，并把缓冲器派送到通道。唯一直接和通道交互的缓冲器是ByteBuffer，它用于以原始的字节形式或基本数据类型输出和读取数据，无法处理字符型对象，这种处理方式低级，但符合大多数操作系统的映射方式。
NIO采用非阻塞型 I/O，实现了异步传输。一个 buffer 主要由 position,limit,capacity 三个变量来控制读写的过程。
对于任何流类，getChannel()将会产生一个FileChannel，打开一个通道，可以向通道传送用于读写的ByteBuffer(通过其静态方法wrap或allocate获得)，并且可以锁定文件的部分区域用于独占式访问。每次read()完后，就会将数据输入到缓冲器中，接着需要调用flip()，该方法把偏移量设置到当前位置，用于接下来的write()，而write()完后需要调用clear()把偏移量设置到容量大小位置用于下次的read。
工作原理：1.由一个专门的线程Selector通道原理器来处理所有的 IO 事件，并负责分发；2. 事件驱动机制：事件到的时候触发，而不是同步的去监视事件；3. 线程通讯：线程之间通过 wait,notify 等方式通讯，保证每次上下文切换都是有意义的，减少无谓的线程切换。
java NIO采用了双向通道（channel）进行数据传输。服务端和客户端各自维护一个管理通道的对象，我们称之为selector，该对象能检测一个或多个通道 (channel) 上的事件。我们以服务端为例，如果服务端的selector上注册了读事件，某时刻客户端给服务端发送了一些数据，阻塞I/O这时会调用read()方法阻塞地读取数据，而NIO的服务端会在selector中添加一个读事件。服务端的处理线程会轮询地访问selector，如果访问selector时发现有感兴趣的事件到达，则处理这些事件，如果没有感兴趣的事件到达，则处理线程会一直阻塞直到感兴趣的事件到达为止。
Selector异步 IO 的核心类，它能检测一个或多个通道 (channel) 上的事件，并将事件分发出去。使用一个 select 线程就能监听多个通道上的事件，并基于事件驱动触发相应的响应，而不需要为每个 channel 去分配一个线程。SelectionKey包含了事件的状态信息和时间对应的通道的绑定。
视图缓冲器(view buffer) 可以让我们通过某个特定的基本数据类型的视窗查看其底层的ByteBuffer，ByteBuffer依然是实际存储数据的地方，支持的前面的视图。
缓冲器的细节：Buffer由数据和可以高效地访问及操纵这些数据的四个索引组成，这四个索引是：mark(标记)、position(位置)、limit(界限)、capacity(容量)。在缓冲器中插入和提取数据的方法会更新这些索引，用于反映所发生的变化，比如clear()方法会把limit设置为capacity，这样下次read()时可以把文件内容读满缓冲器，flip()方法把limit设置为position，接下来write()持久化数据时也保证不会溢界。
内存映射文件 允许创建和修改那些因为太大而不能放入内存的文件。MappedByteBuffer 通过调用通道上的map()方法产生，是一种特殊的直接缓冲器。其原理是先指定映射文件的初始位置和映射区域的长度，先把文件的一部分放入内存中，文件其他部分和已处理过的文件在内存不断交换，最终处理完整个大文件。

二十、注解(Annotate)

注解也称为元数据(MetaData)为我们在代码中添加信息提供了一种形式化的方法，可以在稍后某个时刻非常方便地使用这些数据。
Java内置三种注解：@Override、@Deprecated、@Suppress Warning，并提供了专门负责创建新注解的四种元注解：@Target、@Retention、@Documented、@Inherited。
一般注解的流程是：定义注解->使用注解->处理注解。其中处理注解是通过创建 注解处理器(APT, Annotate Processor Tool)，通过getAnnotation()获取指定的注解对象。
基于注解的单元测试：在Java中，最著名的单元测试工具就是JUnit。
